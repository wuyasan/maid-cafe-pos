# 员工账号 与 账单打折 — 使用说明

这次给系统加了两个功能：一个是**员工账号**，把过去"店员和管理员共用一个 PIN"换成了每个人自己的账号和角色；另一个是**账单打折**，店员可以给某一桌的账单整单打个百分比折扣或者减一个固定金额。下面分别说明怎么用。

## 一、员工账号

### 角色怎么分

系统里有三种角色。**店员（staff）**负责日常运营——点单、后厨吧台出品、跑餐、收款，也可以打折。**经理（manager）**目前能做的和店员一样；这个角色是为后面要做的对账、审计、退款预留的，所以现在它还进不了后台管理区。**管理员（admin）**权限最大，除了店员能做的，还能进后台（`/admin`）管理菜单、场次、桌台、女仆、定价，以及管理员工账号本身。

页面层面：所有 `/staff` 下的页面只要登录了就能进；`/admin` 下的页面只有管理员能进。非管理员登录后，侧边栏和 `/staff` 首页都不会显示"后台"入口，不会出现"点进去却被踢回登录页"的情况。

### 怎么登录

打开应用会停在登录页。输入**用户名 + PIN** 就能登录，成功后进入店员工作台。如果用户名或 PIN 错了，页面只会提示"用户名或 PIN 错误"，不会告诉你具体是哪一项错——这是有意的，避免被人试探。连续输错会被限流，挡一阵子防止有人暴力猜 PIN。

### 管理员怎么管账号

用管理员登录后，进**后台 → 员工账号**（`/admin/staff-users`）。在这里可以：新建一个员工（填用户名、显示名、角色、PIN）；修改某人的显示名和角色（用户名一旦创建就不能改）；停用或重新启用某个账号（被停用的人无法登录）；以及给某人重置 PIN。

为了不让脏数据进库，后端会强制几条规则（前端表单也会提示，但真正把关的是后端）：PIN 必须是 4 到 12 位数字；用户名 3 到 50 位、只能用小写字母数字和 `_ . -`（系统会自动去空格并转小写）；显示名不能为空。另外有两条保护：**最后一个还在启用的管理员不能被停用或降级**，这样你不会把自己锁在系统外面；用户名重复会被拒绝，即使两个人同时创建同名账号也不会出错。

### 第一次启动 / 初始账号

后端启动时，如果数据库里还没有对应账号，会根据环境变量自动建几个初始账号：`ADMIN_PIN` 建管理员 `admin`、`MANAGER_PIN` 建经理 `manager`、`STAFF_PIN` 建店员 `staff`。本地开发默认是 `admin/9999`、`manager/8888`、`staff/1234`。这个过程是幂等的（重复启动不会重复建），也可以用 `ENABLE_STARTUP_BOOTSTRAP=false` 关掉。

**正式上线前请务必**：改掉这几个默认 PIN（或者建好真实账号再把种子账号停用）、设置一个足够强的 `AUTH_SECRET`、用 HTTPS 提供服务（登录 cookie 在生产环境是 Secure 的），并在受控反向代理后面打开 `TRUST_PROXY_HEADERS=true`，这样按 IP 的登录限流才能正常工作。

### 审计地基

现在前端对后端发起的每一个写操作，都会带上当前操作人的身份（`X-Actor-Id` / `X-Actor-Role`）。目前界面上还看不到这些信息，但它是为接下来的"操作审计日志"功能埋好的钩子——到时候就能查"谁在什么时候做了什么"。

## 二、账单打折

可以给某一桌**还没结账（open 状态）**的账单整单打折。注意是整单折扣，不是针对单个菜品。

### 怎么打折

从楼面点进桌台，或直接打开 `/staff/table/<桌号>`，在账单区点 **Apply discount（打折）**，然后选择类型并填数值：选**百分比**就是按比例打折，比如填 `10` 表示打 9 折（范围 0–100）；选**固定金额**就是直接减钱，比如填 `5` 表示减 5 元。还可以填一条备注（自由文本，最多 500 字），比如"常客""生日"。确认之后，账单就会显示成**小计 → 折扣 −$X（带备注）→ 应付**三行。想改就点"编辑折扣"，想取消就点"移除折扣"。

### 规则和行为

**谁能打折**：任意店员都可以打，不设上限——靠记录来兜底，每一笔折扣都会记下是谁打的（操作人）以及备注，方便事后追责。

**金额一律以后端为准**：应付总额永远由后端重新算，公式是 `应付 = max(0, 小计 − 折扣) + 税 + 服务费`（税和服务费目前都是 0）。

**百分比折扣会自动跟着重算**：之后加菜、减菜、改数量，10% 始终按当前小计算，不会算错。**固定金额折扣会被限制在小计以内**：比如一张 30 元的单打了 50 元的固定折扣，应付会变成 0，而不会出现负数。

**只有 open 账单能改折扣**：一旦开始结账（paying）或已经付款（paid），打折相关的按钮就会禁用，硬调接口会返回明确的错误。如果一张已经在结账中的单要打折，得先取消结账再打。

**收款用的是折后价**：开始结账时会把折后的金额冻结下来，之后无论走 Square 还是手动标记已付，扣的都是折后这个数。

**异常输入会被挡住、不会让后端崩**：备注超过 500 字、百分比超过 100、负数或非数字，都会返回清晰的报错，而不是 500。

### 顾客那边看到什么

当账单有折扣时，顾客的点单/账单页（`/order/<桌号>`）会在应付总额上方，只读地显示出小计和折扣这两行，让顾客也清楚便宜了多少。

## 配置项速查

- `AUTH_SECRET`（web）：给登录 cookie 签名，生产必填。
- `ADMIN_PIN` / `MANAGER_PIN` / `STAFF_PIN`（后端）：首次启动时种初始账号。
- `ENABLE_STARTUP_BOOTSTRAP`（后端）：设为 `false` 可跳过种账号，默认开。
- `TRUST_PROXY_HEADERS`（web）：在受控反向代理后设为 `true`，让登录限流按真实客户端 IP 生效。
- `INTERNAL_GATEWAY_TOKEN`（web + 后端）：web 调后端时带的共享令牌，空字符串/纯空格视为未配置。

## 已知限制

- 后端目前只用 gateway 令牌做整体鉴权，还没有做到按用户逐人鉴权——角色限制是在 web 层强制的。把鉴权下沉到 API 逐用户，是后续要补的。
- `manager` 角色暂时还没有专属页面，是为后面的对账、审计、退款功能预留的。
